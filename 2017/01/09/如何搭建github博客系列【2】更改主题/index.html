如何更改自带的主题？

2017-01-09

  1. 1. 前言
  2. 2. 了解Hexo的文件结构
  3. 3. 下载主题
  4. 4. 启用主题

前言

如果不喜欢自带的主题怎么办？我们可以自己尝试更换它。本文以Tinnypp主题为例，该主题是我从github上fork下来的。

了解Hexo的文件结构

当然，更多的还是需要仔细阅读文档才能了解更详细

_config.yml 全局配置文件。要注意的是，该文件格式要求极为严格，缺少一个空格都会导致运行错误。小提示：不要用Tab缩进，两个空格符， 冒号：后面只用一个空格即可 。
themes 存放主题的文件夹
source 博客资源文件夹
source/_drafts 草稿文件夹
source/_posts 文章文件夹
themes/landscape 默认皮肤文件夹
…

下载主题

建立了Hexo文件之后就可以下载主题了，在themes文件夹下使用git命令：

$ git clone https://github.com/A1bertLee/Tinnypp

上面的是https格式的链接，ssh也可以。

启用主题

  • 修改参数 修改Hexo目录下的config.yml(Hexo有两个config.yml文件，一个在根目录，一个在theme下，此时修改的是在根目录下的)配置文件中的theme属性，将其设置为Tinnypp（相应的主题名字）。
    同时请设置stylus属性中的compress值为true。

    theme: Tinnypp

  • 注意： 为避免出错，请先备份你的_config.yml 文件后再升级。

  • 本地查看调试

    $ hexo clean #清理之前生成html文件
    $ hexo g #生成html文件
    $ hexo s #启动本地服务，进行文章预览调试

浏览器输入：http://localhost:4000 查看搭建效果。此后的每次变更_config.yml文件或者上传文件以及改动没有效果，都可以先这三个命令调试，非常好用，尤其是当你想调试出自己想要的主题时。
